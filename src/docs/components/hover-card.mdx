# HoverCard

A popup card that appears when hovering over a trigger element.

## Usage

```tsx
import { HoverCard } from "@/components/HoverCard/HoverCard";

export default function HoverCardDemo() {
  return (
    <HoverCard trigger={<Button>Hover me</Button>}>
      <div className="space-y-2">
        <h4 className="font-medium">Preview Content</h4>
        <p className="text-sm text-muted-foreground">
          Additional information that appears on hover.
        </p>
      </div>
    </HoverCard>
  );
}
```

## Props

- `trigger` - Element that triggers the hover card
- `openDelay` - Delay before showing (default: 200ms)
- `closeDelay` - Delay before hiding (default: 300ms)
- `side` - Position relative to trigger
  - `top`
  - `right`
  - `bottom` (default)
  - `left`
- `align` - Alignment along the chosen side
  - `start`
  - `center` (default)
  - `end`
- `arrow` - Show arrow pointer (default: true)
- `className` - Additional CSS classes

```tsx

</HoverCard>
```

### Without Arrow

```tsx

</HoverCard>
```

```tsx
</HoverCard>
```

## Accessibility

- Uses proper ARIA attributes
- Supports keyboard focus management
- Handles mouse and touch interactions
- Provides sufficient hover delay
- Maintains proper contrast

## Best Practices

1. Content Guidelines

   - Keep content concise
   - Show relevant preview information
   - Use consistent content structure
   - Avoid complex interactions

2. Timing

   - Use appropriate delays
   - Consider user reading time
   - Balance responsiveness
   - Handle rapid hover changes

3. Positioning

   - Consider viewport edges
   - Maintain visibility
   - Use consistent alignment
   - Handle overflow properly

4. Visual Design

   - Clear visual hierarchy
   - Sufficient contrast
   - Smooth animations
   - Proper spacing

5. Performance
   - Optimize content loading
   - Minimize layout shifts
   - Handle rapid interactions
   - Clean up timeouts
