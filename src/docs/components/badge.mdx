# Badge

Badges are used to highlight status, show counts, or emphasize small pieces of information.

## Usage

```tsx
import { Badge } from "@/components/Badge/Badge";

export default function BadgeDemo() {
  return (
    <div className="flex gap-4">
      <Badge>Default</Badge>
      <Badge variant="secondary">Secondary</Badge>
      <Badge variant="destructive">Destructive</Badge>
    </div>
  );
}
```

## Props

- `variant` - Visual style variant
  - `default` - Primary color theme
  - `secondary` - Secondary color theme
  - `destructive` - For error or warning states
  - `outline` - Bordered style
  - `success` - For success states
  - `warning` - For warning states
- `size` - Size of the badge
  - `sm` - Small
  - `md` - Medium (default)
  - `lg` - Large
- `rounded` - Whether to use fully rounded corners (default: true)
- `className` - Additional CSS classes
- All HTML div attributes are also supported

## Examples

### Variants

```tsx
<div className="flex flex-wrap gap-4">
  <Badge>Default</Badge>
  <Badge variant="secondary">Secondary</Badge>
  <Badge variant="destructive">Destructive</Badge>
  <Badge variant="outline">Outline</Badge>
  <Badge variant="success">Success</Badge>
  <Badge variant="warning">Warning</Badge>

```

### Sizes

```tsx
<div className="flex flex-wrap gap-4 items-center">
  <Badge size="sm">Small</Badge>
  <Badge size="md">Medium</Badge>
  <Badge size="lg">Large</Badge>
</div>
```

### With Icons

```tsx
import { CheckCircle, XCircle, AlertTriangle } from "lucide-react";

<div className="flex flex-wrap gap-4">
  <Badge variant="success" className="gap-1">
    <CheckCircle className="w-3 h-3" />

  </Badge>
  <Badge variant="destructive" className="gap-1">
    <XCircle className="w-3 h-3" />
    Error
  </Badge>
  <Badge variant="warning" className="gap-1">
    <AlertTriangle className="w-3 h-3" />
    Warning
  </Badge>

```

### Corner Styles

```tsx
<div className="flex flex-wrap gap-4">
  <Badge rounded>Rounded</Badge>
  <Badge rounded={false}>Square</Badge>
</div>
```

### Custom Styles

```tsx
<Badge className="bg-purple-500 hover:bg-purple-600 text-white">
  Custom Color
</Badge>
```

## Accessibility

- Use color combinations that meet WCAG contrast requirements
- When using badges to convey status or information, ensure the meaning is not conveyed by color alone
- Consider adding aria-label when the badge content might be ambiguous
- If the badge is purely decorative, add aria-hidden="true"

## Best Practices

1. Usage Guidelines

   - Use badges sparingly to avoid visual clutter
   - Keep text content brief and concise
   - Use consistent variants for similar types of information
   - Consider using icons to reinforce meaning

2. Placement

   - Position badges close to related content
   - Avoid using too many badges in a single view
   - Maintain consistent spacing around badges

3. Variants

   - Use `default` for neutral information
   - Use `success` for positive outcomes
   - Use `destructive` for errors or critical states
   - Use `warning` for cautionary information
   - Use `outline` for less emphasis
   - Use `secondary` for supplementary information

4. Customization
   - Maintain consistent styling across your application
   - Use custom colors sparingly and ensure they align with your design system
   - Consider dark mode compatibility when customizing
